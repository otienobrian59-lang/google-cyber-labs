# Security Policy

## Reporting a Security Vulnerability

This is a documentation-only repository for educational purposes. However, if you discover any security issues within the notes, scripts, or the repository structure itself, please report them as follows:

1. **Do not open a public issue.**
2. Send an email to the repository owner (details in profile).
3. Provide a detailed description of the issue and steps to reproduce.

I will acknowledge your report and work on a fix as soon as possible.

## Security Best Practices for this Repo
- **No Sensitive Data**: I ensure that no passwords, API keys, or PII (Personally Identifiable Information) are ever committed to this repository.
- **Verification**: All commands and scripts are for educational use and should be tested in a controlled environment (like a VM) before use.
